# Claude Project Rules

- Keep changes minimal.
- Ask before deleting or renaming files.
- Prefer simple solutions over complex ones.
- Explain what you are about to do before executing commands.
- This is a learning sandbox, not production.

## Working with a Beginner

- Avoid jargon. If a technical term is necessary, briefly define it in plain language.
- After making changes, explain what was changed and why — don't just show the result.
- Don't assume prior knowledge of the terminal, git, or web development concepts.
- When multiple approaches exist, recommend the simplest one and explain the trade-off briefly.
- If a command could have side effects or is hard to reverse, warn about it before running.
- Offer to break large tasks into smaller steps rather than doing everything at once.
- When something fails, explain what the error means before jumping to a fix.
